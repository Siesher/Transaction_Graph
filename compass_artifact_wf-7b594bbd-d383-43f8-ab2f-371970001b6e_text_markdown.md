# Граф связей компании из транзакций: полная методология

Построение графа корпоративных связей на основе банковских транзакционных данных — зрелая аналитическая дисциплина, объединяющая теорию сетей, машинное обучение на графах и комплаенс-практику. **Ключевой вывод исследования: наиболее эффективный подход — гетерогенный граф с несколькими типами узлов и рёбер, обработанный через Leiden-кластеризацию и GNN-классификацию, с предварительной фильтрацией шума через disparity filter Серрано.** Этот подход позволяет отделить структурные аффилированные связи от обычных контрагентских отношений с точностью, подтверждённой на реальных банковских данных (Sberbank/Skoltech, DNB Norway). Ниже изложена полная методология от построения графа до выявления скрытых бенефициаров.

---

## Как превратить транзакции в граф

Первый и критический шаг — трансформация «плоских» транзакционных записей в структурированный граф. Практика крупных банков (NatWest, Sberbank, DNB) выделяет три архитектуры узлов: **на уровне счетов** (каждый IBAN — узел), **на уровне юридических/физических лиц** (агрегация всех счетов одного клиента) и **гибридная иерархическая** (клиент → счёт → транзакция). Для задачи выявления корпоративных связей оптимален уровень юридических лиц с сохранением физических лиц как отдельного типа узлов — это формирует **гетерогенный граф**, где разные типы узлов (компании, ИП, физлица) соединены разными типами рёбер (платежи, зарплатные выплаты, переводы между счетами).

Рёбра формируются из агрегированных транзакций за временное окно (обычно месяц или квартал). Каждое ребро — направленное (плательщик → получатель) и несёт вектор признаков. Ключевые признаки рёбер, определяющие характер связи:

- **Объём** — суммарный денежный поток w(i,j) = Σ amount(t) за период
- **Частота** — количество транзакций f(i,j) между парой контрагентов
- **Доля в обороте** — share(i,j) = w(i,j) / Σ_k w(i,k), показывающая какую часть исходящего потока узла i составляют платежи в j
- **Взаимность** — R(i,j) = min(w(i→j), w(j→i)) / max(w(i→j), w(j→i)), отношение встречных потоков
- **Регулярность** — коэффициент вариации интервалов между транзакциями; низкий CV указывает на систематические, а не разовые платежи
- **Временная концентрация (burstiness)** — кластеризация транзакций во времени, характерная для схемных операций

Исследование Shumovskaia et al. (2021, Sberbank/Skoltech) показало, что обработка временных рядов транзакций через RNN и формирование эмбеддингов узлов из этих рядов значительно повышает качество предсказания связей. Их модель SEAL-RNN использует рекуррентную сеть для кодирования временных паттернов каждого узла, а затем графовую свёрточную сеть для предсказания наличия связи между парой узлов. На данных крупного европейского банка (миллионы узлов, миллиарды рёбер) модель показала значимое превосходство по ROC AUC над базовыми методами.

Отдельный вопрос — **темпоральная конструкция графа**. Существуют три подхода: снимки (snapshot) с агрегацией за фиксированные окна, формирующие последовательность статических графов G_t; темпоральный мультиграф с сохранением всех отдельных транзакций с метками времени; и потоковая обработка (streaming), где GNN-эмбеддинги обновляются инкрементально при поступлении каждой транзакции (подход ROLAND, Stanford). Для задачи корпоративных связей среднего масштаба оптимальны квартальные снимки с дополнительной историей за 2–3 года.

## Весовые метрики рёбер и борьба с шумом

Трансформация множества транзакций между двумя контрагентами в одно взвешенное ребро — нетривиальная задача, от решения которой зависит качество всей последующей аналитики. **Наиболее информативная метрика — доля в обороте контрагента p_ij = w_ij / s_i**, где s_i — суммарный исходящий поток узла i. Она нормализует абсолютные объёмы и показывает относительную значимость связи для каждого участника. Компания, направляющая **более 30% своего оборота** одному контрагенту, с высокой вероятностью аффилирована с ним — это редко встречается в обычных коммерческих отношениях.

Для агрегации множественных транзакций в вектор признаков ребра применяются: суммарный объём, количество операций, средняя и максимальная сумма, стандартное отклонение, дата последней транзакции. Этот вектор [sum, count, mean, std, max, min, last_date] подаётся на вход GNN. Для алгоритмов, требующих скалярного веса, используют комбинацию w(i,j) = α·volume + β·frequency с нормализацией, либо логарифмическое преобразование w̃ = log(1 + w) для сжатия тяжёлых хвостов распределения. Для учёта актуальности связей применяют экспоненциальное затухание: w(i,j) = Σ_t amount_t · exp(−λ(T−t)).

### Фильтрация шума: от сырого графа к «скелету» связей

Сырой транзакционный граф содержит огромное количество случайных, разовых и нерелевантных связей. Ключевая задача — извлечь «backbone» (скелет) значимых структурных связей. Золотой стандарт — **disparity filter Серрано** (Serrano, Boguñá, Vespignani, PNAS 2009). Алгоритм работает следующим образом: для каждого узла i с k_i связями и суммарной силой s_i нормализуются веса рёбер p_ij = w_ij / s_i; затем вычисляется вероятность наблюдения такого нормализованного веса при равномерном распределении: **α_ij = (1 − p_ij)^(k_i − 1)**; ребро сохраняется, если α_ij < α для хотя бы одного из двух концов. При α = 0.05 фильтр сохраняет рёбра, значимые на локальном уровне каждого узла, что критично для гетерогенных сетей, где крупные хабы имеют много слабых, но важных связей.

Для зашумлённых данных (неполные транзакции, ошибки сумм) предпочтителен **Noise-Corrected backbone** (Coscia & Neffke, 2017), использующий байесовский подход с учётом склонности обоих концов ребра. Ожидаемый вес моделируется как E[w_ij] = s_i^out · s_j^in / W, а отклонения оцениваются через апостериорную дисперсию.

Рекомендуемый пайплайн фильтрации для банковского графа:

1. Предобработка: удаление рёбер с числом транзакций < 3 и суммарным объёмом ниже порога
2. Темпоральный фильтр: связь должна присутствовать минимум в 2 из N временных окон
3. Backbone extraction: disparity filter (α = 0.05) для чистых данных или noise-corrected для зашумлённых
4. Валидация: сравнение извлечённого скелета с известными корпоративными структурами

## Алгоритмы на графах: от кластеризации до нейросетей

### Community detection для кластеризации аффилированных структур

**Leiden** (Traag, Waltman, van Eck, 2019) — рекомендуемый основной алгоритм. Он улучшает классический Louvain тремя существенными способами: гарантирует связность найденных сообществ (Louvain порождает до 25% плохо связанных и 16% разорванных кластеров); добавляет фазу рефайнмента, разделяющую кластеры на хорошо связанные части; и сходится быстрее. Для корпоративных сетей важно использовать **Constant Potts Model (CPM)** вместо стандартной модулярности: H = Σ_c [e_c − γ·C(n_c, 2)], где γ — параметр разрешения. CPM свободен от «resolution limit» — проблемы, при которой модулярность не может выделить сообщества меньше определённого масштаба.

**Infomap** (Rosvall & Bergstrom, 2008) — лучший выбор для анализа денежных потоков. Алгоритм минимизирует описание случайного блуждания по сети (map equation), естественно учитывая направление и вес рёбер. Для корпоративного графа это означает, что Infomap выделяет группы, внутри которых деньги циркулируют интенсивнее, чем между группами — именно так устроены холдинговые структуры. Infomap также находит иерархическую вложенную структуру сообществ, что соответствует реальной иерархии «головная компания → дивизионы → дочерние общества».

**Практическая стратегия**: запуск Leiden с CPM при нескольких значениях γ (0.5, 0.8, 1.0, 1.5, 2.0) и перекрёстная валидация с Infomap. Устойчивые кластеры, воспроизводимые обоими алгоритмами, с высокой вероятностью соответствуют реальным корпоративным группам.

### Link prediction для обнаружения скрытых связей

Предсказание связей позволяет обнаружить аффилированность между компаниями, которые не транзачат напрямую, но связаны через общего бенефициара. Классические топологические метрики работают как базовые детекторы: **Adamic-Adar index** AA(x,y) = Σ_{z ∈ Γ(x)∩Γ(y)} 1/log|Γ(z)| взвешивает общих контрагентов обратным логарифмом их степени — редкий общий контрагент (например, узкоспециализированная компания) даёт более сильный сигнал аффилированности, чем общий крупный банк. **Индекс Катца** Katz(x,y) = Σ β^ℓ · |paths_xy^(ℓ)| суммирует все пути между узлами с затуханием по длине и показывает лучшую производительность в сравнительных исследованиях.

Для более глубокого анализа применяются **графовые эмбеддинги**. Node2Vec (Grover & Leskovec, 2016) генерирует вектрные представления узлов через смещённые случайные блуждания с параметрами p (возврат) и q (исследование). При низком q модель захватывает гомофилию — компании одной группы получают близкие эмбеддинги. При низком p — структурную эквивалентность, позволяющую находить компании в похожих ролях (например, несколько «прокладок» одного холдинга). Компании со сходными эмбеддингами (по косинусной мере) и отсутствием прямых транзакций — кандидаты на скрытую аффилированность.

### Graph Neural Networks: GCN, GraphSAGE, GAT

GNN представляют наиболее мощный инструмент для классификации узлов и рёбер в корпоративном графе. Систематический обзор Abdallah et al. (2024, Expert Systems with Applications), охватывающий более 100 исследований, подтверждает: **GNN значимо превосходят традиционные методы, поскольку захватывают реляционные паттерны**, а не только признаки изолированных транзакций.

**GCN** (Kipf & Welling, 2017) выполняет спектральные свёртки по графу: H^(l+1) = σ(D̃^{-1/2} Ã D̃^{-1/2} H^(l) W^(l)). Для финансовых графов GCN служит базовой архитектурой, но имеет ограничения: трансдуктивный подход (не обобщается на новые узлы) и проблемы с классовым дисбалансом (аффилированные рёбра — менее 1% всех рёбер).

**GraphSAGE** (Hamilton et al., 2017) решает проблему масштабируемости через индуктивное обучение: модель учит функцию агрегации соседей, а не фиксированные эмбеддинги. Практическое внедрение в NatWest: mean-агрегатор с uniform random sampling соседей, hidden_channels=16, n_hops=1, batch_size=1024. GraphSAGE эффективно нейтрализует влияние суперсвязанных узлов (крупные розничные контрагенты) и обрабатывает миллионы узлов через мини-батчи.

**GAT** (Veličković et al., 2018) добавляет механизм внимания: α_ij = softmax(LeakyReLU(a^T[Wh_i || Wh_j])), позволяя модели автоматически определять, какие связи наиболее информативны. Для банковских транзакций расширения GAT (EWS-GCN от Sberbank) включают веса рёбер непосредственно в коэффициенты внимания, а Minakawa & Sakaji добавляют текстовую отраслевую информацию, повышая качество на **3.1%**.

Наиболее релевантная работа — **Johannessen & Jullum (2023)**, первое опубликованное применение GNN на реальном гетерогенном банковском графе DNB (крупнейший банк Норвегии). Они расширили MPNN для работы с несколькими типами узлов (клиенты, счета, компании) и рёбер (транзакции, владение, роли), агрегируя сообщения по разным типам связей отдельно.

Для задачи классификации рёбер как «аффилированных» vs «обычных коммерческих» оптимальна архитектура из **2–3 слоёв GraphSAGE или GAT** с включением признаков рёбер. Обучение — полусупервизированное: менее 5% рёбер размечены (из корпоративных реестров, раскрытой информации об аффилированности), GNN распространяет метки через структуру графа.

### Centrality measures для определения ролей

Меры центральности раскрывают роли узлов в корпоративной структуре. **PageRank** в транзакционной сети выявляет компании, получающие значительные потоки от других важных компаний — это паттерн головной/материнской компании, аккумулирующей дивиденды и управленческие платежи от дочерних структур. Взвешенный PageRank: PR_w(i) = (1−d)/N + d · Σ_{j→i} (w_ji/Σ_k w_jk) · PR_w(j).

**Betweenness centrality** — ключевая метрика для обнаружения «прокладок». Компания-прокладка лежит на множестве кратчайших путей между реальными бизнес-сущностями, обеспечивая проход денежных потоков. Диагностический паттерн: **высокая betweenness при низком собственном обороте** — компания не ведёт реальной деятельности, но критична для маршрутизации платежей. Формально: Suspicion(v) = α·norm(BC(v)) + β·norm(1/ClusterCoef(v)) + γ·(BC(v)/strength(v)).

Профили типичных узлов по центральности:

| Тип сущности | PageRank | Betweenness | Clustering Coef. | Степень |
|---|---|---|---|---|
| Материнская компания | Высокий | Умеренный | Средний | Средняя, высокий in-degree |
| Прокладка/shell | Низкий | Очень высокий | Близок к 0 | Средняя, fan-in/fan-out |
| Операционная «дочка» | Умеренный | Низкий | Средний-высокий | Высокая (много контрагентов) |
| Финансовый кондуит | Низкий | Высокий | Низкий | Низкая, мост между группами |

## Как отличить аффилированность от обычного контрагентства

Центральный методологический вопрос — разграничение нормальных коммерческих связей и связей типа «материнская-дочерняя компания» или скрытой аффилированности. Транзакционные паттерны, свидетельствующие об аффилированности:

**Циркулярные платежи** — денежный поток A → B → C → A, выявляемый через алгоритмы поиска циклов (DFS, алгоритм Джонсона) с темпоральной фильтрацией: нелегитимные циклы замыкаются за дни, легитимные (цепочки поставок) — за недели и месяцы. **Синхронность операций** — связанные компании показывают коррелированные по времени дебетовые и кредитовые операции. **Постоянство пропорций** — стабильный процент оборота, направляемый одному контрагенту (например, всегда 30%), указывает на управленческое, а не рыночное ценообразование. **Концентрация контрагентов** — непропорционально высокая доля операций с узкой группой компаний.

Работа Mao et al. (2022) напрямую адресует эту задачу: строится гетерогенный граф из листинговых компаний и их связанных сторон (акционеры, директора, контрагенты), на котором GNN предсказывает мошеннические связанные транзакции. Модель значимо превосходит традиционные подходы, использующие лишь агрегированную статистику связанных операций.

Для выявления **shell-компаний** характерен паттерн «money in ≈ money out»: flow-through ratio = min(total_in, total_out) / max(total_in, total_out) близок к 1.0. Дополнительные маркеры: отсутствие зарплатных выплат и налоговых платежей, минимальные операционные расходы, пакетная активность с периодами «спячки». В графе shell-компании отличаются высокой betweenness centrality при низком clustering coefficient — их контрагенты не связаны друг с другом, что нетипично для реального бизнеса. Исследование на данных UK Companies House (Tiwari, Neo4j) с использованием графовых алгоритмов показало **87–97% точность** обнаружения shell-компаний.

Ключевое исследование Lőrincz et al. (2023, Network Science) продемонстрировало на мультислойной сети (слой транзакций + слой совместного владения), что **связи владения сильно предсказывают транзакционные связи**, и наоборот. Прямые и непрямые связи в обоих слоях прогнозируют создание и поддержание экономических транзакций, что позволяет реконструировать скрытую аффилированность из транзакционных данных.

## Применение в AML/комплаенс и российская специфика

Графовая аналитика стала отраслевым стандартом для продвинутого AML, вытесняя правила-based системы с уровнем ложных срабатываний **~95%**. FATF в Рекомендации 24 (пересмотрена в 2022) требует «многоуровневого подхода» к идентификации бенефициарных владельцев, объединяющего реестры, данные компаний и данные финансовых институтов — это прямо соответствует методологии гетерогенного графа с несколькими источниками данных.

Базельский Принцип 20 (CP20, пересмотрен в 2024) усилил требования к идентификации связанных сторон, одобрению операций с ними и отчётности. IMF Technical Note (2024) формализует диагностику банков через предопределённые индикаторы юридических, управленческих, операционных и экономических связей.

В России 115-ФЗ определяет бенефициарного владельца как физическое лицо, прямо или косвенно владеющее **более 25%** капитала клиента или способное контролировать его действия. Платформа ЦБ РФ «Знай своего клиента» (ФЗ-479) присваивает клиентам уровни риска (зелёный/жёлтый/красный), что может использоваться как дополнительный признак узлов в графе. Положение 375-П содержит коды необычных операций, которые естественно транслируются в правила на графе: отсутствие зарплатных выплат при наличии оборота, транзитный характер операций, круговые платежи между связанными лицами.

Для обнаружения схем на графе используются специализированные алгоритмы: **FlowScope** (Li et al., 2020) детектирует аномальные денежные потоки в многослойных транзакционных графах с гарантированными границами обнаружения; **SMoTeF** (2024) расширяет обнаружение смёрфинга через вычисление максимального темпорального потока; **GARG-AML** (2025) оценивает клиентов по их окружению второго порядка в графе.

## Ключевые академические работы и их вклад

Фундамент методологии формируют несколько групп исследований. **Vitali, Glattfelder & Battiston (2011, PLoS ONE)** — семинальная работа, построившая глобальную сеть корпоративного контроля из 43 000 ТНК и обнаружившая «суперсущность» из 147 тесно связанных компаний, контролирующих 40% богатства сети. **Shumovskaia et al. (2021, Sberbank/Skoltech)** — наиболее прямо релевантная работа: GNN link prediction на банковском транзакционном графе с RNN-обработкой временных рядов; модель SEAL-RNN валидирована на данных крупного банка. **Johannessen & Jullum (2023)** — первое опубликованное применение гетерогенной GNN на реальных данных для AML (банк DNB, Норвегия). **Mao et al. (2022, Procedia Computer Science)** — GNN на графе связанных сторон для обнаружения мошеннических RPT-транзакций. **Weber et al. (2019, KDD Workshop)** — эксперименты с GCN для AML на Elliptic Dataset, ставшем де-факто бенчмарком в области.

Методологические основы: **Traag et al. (2019)** — алгоритм Leiden; **Serrano et al. (2009, PNAS)** — disparity filter; **Coscia & Neffke (2017)** — noise-corrected backbone; **Rosvall & Bergstrom (2008)** — Infomap; **Grover & Leskovec (2016)** — Node2Vec. Обзорные работы: **Abdallah et al. (2024, Expert Systems with Applications)** — систематический обзор 100+ работ по GNN для финансового мошенничества; **Deprez et al. (2025)** — обзор continual graph learning для AML; **Egressy et al. (2023)** — GNN для направленных мультиграфов с применением к финансовым преступлениям.

Для генерации синтетических данных и тестирования: **IBM AMLSim** — мультиагентный симулятор банковских транзакций с паттернами отмывания (smurfing, fan-in/fan-out, циклы); **Altman et al. (2023)** — генератор синтетических финансовых транзакций для сравнения AML-моделей.

## Итоговая архитектура и практические выводы

Оптимальная архитектура системы для компании среднего уровня (до 10 млрд оборота) включает последовательность этапов, каждый из которых обоснован конкретными исследованиями. Построение гетерогенного графа из транзакционных данных с обогащением из ЕГРЮЛ и KYC-досье. Фильтрация шума через disparity filter с α = 0.05. Кластеризация через Leiden (CPM) с перекрёстной валидацией Infomap. Профилирование узлов через взвешенный PageRank и betweenness centrality. Классификация рёбер через GraphSAGE или GAT с полусупервизированным обучением на известных аффилированных связях. Link prediction через Node2Vec-эмбеддинги для обнаружения скрытых связей.

Принципиальный методологический вывод: **транзакционных данных достаточно для реконструкции значительной части корпоративной структуры**, но наивысшее качество достигается при интеграции с данными о владении (ЕГРЮЛ, Orbis), KYC-информацией (общие директора, адреса, бенефициары) и внешними источниками (санкционные списки, реестры ЦБ). Работа Lőrincz et al. количественно подтвердила: мультислойный граф (транзакции + владение) предсказывает связи значимо лучше, чем каждый слой по отдельности. Для банка с доступом ко всем этим данным методология обеспечивает практически полное покрытие реальных корпоративных связей клиента, включая те, что намеренно скрыты через номинальных владельцев и shell-структуры.
