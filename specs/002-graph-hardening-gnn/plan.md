Harden the Transaction Graph MVP with comprehensive tests, bug fixes, performance optimization, and a pipeline orchestrator. Add GNN-based node classification (R-GCN via DGL) for shell company detection and role classification as a complement to existing heuristic analysis.

## Design Decisions

### D1: enrich_graph placement

`enrich_graph()` goes in `graph_builder.py` because it modifies graph node attributes — same responsibility as `build_graph()` and `compute_edge_metrics()`.

### D2: Pipeline return type

`PipelineResult` is a `namedtuple` (not dataclass) for Python 3.8 compatibility without `__future__` annotations for optional fields.

### D3: GNN module isolation

`gnn.py` is a standalone module with DGL/torch imports guarded by try/except. If DGL is not installed, the module raises `ImportError` with a helpful message. All other modules work without GNN dependencies.

### D4: Test fixtures

`conftest.py` provides session-scoped fixtures that generate synthetic data once and build graph/metrics once per test session. Individual test files use these fixtures.

### D5: Vectorization backward compatibility

Vectorized `classify_node_roles` and `detect_shell_companies` must produce bit-identical results to the old row-by-row versions. Tests compare both implementations on the same data.

### D6: is_liquidated derivation

Node `is_liquidated` is computed as: `deleted_flag == 'Y'` OR (`end_date` is not null AND `end_date < today`). This matches the logic already used in `etl.py` for `client_status_name` but was missing from `graph_builder.py`.
